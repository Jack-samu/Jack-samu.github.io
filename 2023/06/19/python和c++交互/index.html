一、传统的python和c++交互方式

在windows里面，动态库并不是.so文件，而是dll，而且它们需要一个lib文件作为引导，也就是说windows中的动态库是一份lib加上一份dll(针对windows下的动态库可以用nm查看到，符号表在lib中，对应实现其实应该是在dll中才对)。然后，在python中的调用，也是ctypes，不过，是另外的CDLL。这里不对步骤做过多的记录，因为前面有介绍的另一篇推文中有比较详细的介绍，所以，略了略了。

如上，在windows中是可以直接用c++库的函数名的，因为llvm和GNU的区别，如果进行不同编译器编程的，是需要考虑这些细致的区别的。

三、python和c++直接交互

前面的交互，都是建立在最终一个进程的情况，但很多时候需要的是多进程间协作，就比如有一个c++构建的软件跑起来的进程，和一个python跑起来的进程，它们一个负责台面上的接口，一个负责底层的数据，这种情况又应该怎么交互呢？这就是进程间通信的一个应用了。

进程间通信方式，比较普遍的，就是socket通信、管道、消息队列、共享内存和信号量以及信号。

3.1 通过unix域套接字进行通信

在网络编程中，socket是重要的底层支持，无论是什么语言，拉到底层一看，都有着socket的底层调用，而socket网络通信，也就是分处两个PC的两个不同进程间的通信，说白了，也是进程间通信。针对一台机器内的进程间通信呢？一样可以使用socket进行，不过这种socket，叫unix域socket，它支持的是单机内不同进程的通信。

unix domain socket相对网络socket，它不需要经过网络协议栈，不用打包拆包等等协议约定的步骤，它就纯粹把应用层数据从一个进程拷贝到另一个进程，因为是IPC机制的一种实现，所以它也属于ipc的一种。下面是一个python中的unix通信的例子：
